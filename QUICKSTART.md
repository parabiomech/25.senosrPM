# 빠른 시작 가이드

## 1단계: 데이터 수집 📱

### SensorLogger 앱 설정
1. SensorLogger 앱 실행
2. 녹음 시작 버튼 클릭
3. 휠체어에 휴대폰 고정 (화면이 위를 향하도록)

### 테스트 수행
각 테스트마다 시작과 끝에 Annotation 버튼을 눌러주세요.

#### 테스트 1: 원스트로크 푸시
1. "My Annotation 1" 클릭 (시작)
2. 직진 푸시 수행
3. "My Annotation 2" 클릭 (종료)

#### 테스트 2: 회전
1. "My Annotation 1" 클릭 (시작)
2. 회전 동작 수행
3. "My Annotation 2" 클릭 (종료)

#### 테스트 3: 휠리
1. "My Annotation 1" 클릭 (시작)
2. 앞바퀴 들기 → 유지 → 내리기
3. "My Annotation 2" 클릭 (종료)

### 녹음 종료
- 모든 테스트 완료 후 녹음 정지
- 데이터 내보내기

---

## 2단계: 데이터 분석 💻

### 파일 준비
다음 5개 파일이 필요합니다:
- ✅ Accelerometer.csv
- ✅ Gyroscope.csv
- ✅ Orientation.csv
- ✅ Annotation.csv
- ✅ Metadata.csv

### 웹 앱 실행
1. `index.html` 파일을 브라우저에서 열기
2. 또는 Live Server로 실행

### 파일 업로드
1. "CSV 파일 선택 또는 드래그" 영역 클릭
2. 5개 파일 모두 선택 (Ctrl+클릭 또는 Shift+클릭)
3. 업로드 완료 대기

---

## 3단계: 결과 확인 📊

### 테스트 모드 선택
왼쪽 패널에서 분석하고 싶은 테스트 선택:
- ➡️ 직진 푸시
- 🔄 회전
- ⚖️ 휠리

### 분석 결과 확인
각 테스트별로 다음 정보 제공:
- 분석 구간 시간
- 주요 성능 지표
- 측정 정보

### 3D 시각화 및 그래프
- 중앙: 3D 휠체어 움직임 시각화
- 하단: 센서 데이터 그래프
- 재생 컨트롤로 시간대별 확인 가능

---

## 문제 해결 🔧

### Annotation 이벤트가 6개가 아닌 경우
- 각 테스트마다 정확히 시작/종료 2번씩 클릭했는지 확인
- Annotation.csv 파일을 열어서 6개 행이 있는지 확인

### 파일 업로드 오류
- 파일 이름에 "Accelerometer", "Gyroscope" 등의 키워드가 포함되어 있는지 확인
- CSV 형식이 올바른지 확인

### 분석 결과가 이상한 경우
- 휴대폰이 올바른 방향으로 고정되었는지 확인 (화면이 위)
- Annotation 시점이 정확한지 확인
